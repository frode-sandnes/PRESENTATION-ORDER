Web Content Visibility Analysis Examples

Presentation order playground. As documented in:

Frode Eika Sandnes, A Review of How Web Content Order and Omission Unintentionally Can Vary for Different User Cohorts, In: Proceedings of HCI International 2024, Springer, (2024)

Several of the examples are triggered by activating strong zoom/magnification in the browser. The zoom will cause elements to disappear and/or change order.

  • plain: Small and simple document with no issues (reference).

Content order

  • floating: Document with floating elements.
  • force-swap-top-botto: Document with hard-coded positions in opposite order.
  • height-swap: Box order depends on window height.
  • width-swap: Box order depends on window width.
  • flexbox-order: Flexbox order (swap) set in the style definition.
  • flex-wrap-reverse: Flexbox wrap-reverse (swap) depending on width.
  • css-grid-template: Css grid template order specification (swao) set in the style definition.
  • mediatype-order: Use media type width to set flexbox order (swao) set in the style definition.
  • css-grid-media-query: Use media type width in px to set grid layout (swao) with five "zones".
  • css-grid-media-query-em: Use media type width in em to set grid layout (swao) with five "zones".
  • css-grid-auto: CSS grid with auto template, no problems.
  • hyphenation: breaking up a text passage due to long word.

Content omission (hiding)

  • multi-page-plain: A longer document with horizontal scroll.
  • multi-page-fixed: A longer document with element fixed to top.
  • multi-page-sticky: A longer document with sticky element.
  • overshadowing: One box placed on top of another. First on top.
  • overshadowing-default: One box placed on top of another. No z-index settings.
  • overshadowing-reverse: One box placed on top of another. Reversed z-index - second on top.
  • overshadowing-horizontal: One box placed on top of another if width is too narrow.
  • overshadowing-partial: One box is partially placed on top of another.
  • outside-view: One box is positioned outside the viewport so that it becomes invisible.
  • zero-contrast: The same text and background colours makes the text visually imperceivable.
  • visibility-selector: The first box is hidden with the visibllity selector.
  • display-none: Hiding the first box with "display: none".
  • opacity: Hiding the first box with opacity.
  • soft-hyphenation: breaking the long word with soft hyphenation gives no problems.
  • Content in the console.: content is only visible from the console window.
  • Content in html comment.: content is only visible by viewing the page source.

by Frode Eika Sandnes, Oslo Metropolitan University, December, 2023.
